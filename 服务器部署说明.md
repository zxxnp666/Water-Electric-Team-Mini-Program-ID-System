# 🚀 水电班仓库管理系统 - 服务器部署说明

## 📋 系统概述

### 系统架构
- **后端API服务**: `warehouse_backend.py` (端口5000) - 为微信小程序提供接口
- **管理端Web**: `admin_web_app.py` (端口8081) - PC端管理界面
- **MySQL数据库**: 端口3306 - 数据存储
- **微信小程序**: 工人端应用

### 技术栈
- Python 3.8+ + Flask 2.3.3
- MySQL 5.7+
- HTML/CSS/JavaScript

---

### 硬件
- CPU: 2核心+
- 内存: 4GB+ (推荐8GB)

- 网络: 公网IP或固定内网IP

### 软件
- 操作系统: CentOS 7+, Ubuntu 18.04+, Windows Server 2016+
- Python 3.8+
- MySQL 5.7+
- 开放端口: 5000, 8081, 3306

## 🚀 部署步骤

### 1️⃣ 安装基础环境

```bash

sudo yum install python3 python3-pip

# 安装Python (Ubuntu)
sudo apt update
sudo apt install python3 python3-pip

# 安装MySQL (CentOS)
sudo yum install mysql-server
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 安装MySQL (Ubuntu)
sudo apt install mysql-server
sudo systemctl start mysql
sudo systemctl enable mysql

# 验证
python3 --version
mysql --version
```

### 2️⃣ 上传项目文件


```bash

mysql -u root -p

mysql -u warehouse_user -p warehouse_db < sql_statements.sql
mysql -u warehouse_user -p warehouse_db < warehouse_1153_materials.sql

mysql -u warehouse_user -p warehouse_db -e "SHOW TABLES;"
```

## ▶️ 启动服务

### 方式1: 使用生产启动脚本（推荐）
```bash
cd /opt/warehouse_system
python3 production_start.py
```

### 方式2: 后台启动
```bash
# 启动后端
nohup python3 warehouse_backend.py > backend.log 2>&1 &

# 启动管理端
nohup python3 admin_web_app.py > admin.log 2>&1 &

# 查看日志
tail -f backend.log
tail -f admin.log
```

### 方式3: systemd服务（生产推荐）

**创建后端服务** `/etc/systemd/system/warehouse-backend.service`:
```ini
[Unit]
Description=Warehouse Backend API
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/warehouse_system
ExecStart=/opt/warehouse_system/venv/bin/python3 warehouse_backend.py
Restart=always
StandardOutput=append:/var/log/warehouse/backend.log
StandardError=append:/var/log/warehouse/backend_error.log

[Install]
WantedBy=multi-user.target
```

**创建管理端服务** `/etc/systemd/system/warehouse-admin.service`:
```ini
[Unit]
Description=Warehouse Admin Web
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/warehouse_system
ExecStart=/opt/warehouse_system/venv/bin/python3 admin_web_app.py
Restart=always
StandardOutput=append:/var/log/warehouse/admin.log
StandardError=append:/var/log/warehouse/admin_error.log

[Install]
WantedBy=multi-user.target
```

**启用服务**:
```bash
sudo mkdir -p /var/log/warehouse
sudo systemctl daemon-reload
sudo systemctl enable warehouse-backend warehouse-admin
sudo systemctl start warehouse-backend warehouse-admin
sudo systemctl status warehouse-backend warehouse-admin
```

### 数据库备份
```bash
# 手动备份
mysqldump -u warehouse_user -p warehouse_db > backup_$(date +%Y%m%d).sql

# 定时备份脚本
cat > /opt/warehouse_system/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/backup/warehouse"
mkdir -p $BACKUP_DIR
mysqldump -u warehouse_user -p'your_password' warehouse_db | gzip > "$BACKUP_DIR/warehouse_$(date +%Y%m%d_%H%M%S).sql.gz"
find $BACKUP_DIR -name "*.gz" -mtime +30 -delete
EOF

chmod +x /opt/warehouse_system/backup.sh

# 添加到crontab (每天2点备份)
echo "0 2 * * * /opt/warehouse_system/backup.sh" | crontab -
```

## ❗ 故障排除

### 问题1: 服务无法启动
```bash
# 检查Python版本
python3 --version

# 检查依赖
pip list | grep -E 'Flask|mysql-connector'

# 重新安装依赖
pip install -r requirements.txt

# 检查端口占用
netstat -tulnp | grep 5000
kill -9 <PID>  # 杀掉占用进程
```

### 问题2: 数据库连接失败
```bash
# 检查MySQL服务
sudo systemctl status mysql

# 测试连接
mysql -u warehouse_user -p -h localhost

# 检查用户权限
mysql -u root -p -e "SELECT user,host FROM mysql.user WHERE user='warehouse_user';"

# 重新授权
mysql -u root -p << EOF
GRANT ALL PRIVILEGES ON warehouse_db.* TO 'warehouse_user'@'localhost';
FLUSH PRIVILEGES;
EOF
```

### 问题3: 防火墙阻止访问
```bash
# 检查防火墙状态
sudo firewall-cmd --list-ports  # CentOS
sudo ufw status                 # Ubuntu

# 临时关闭防火墙测试
sudo systemctl stop firewalld   # CentOS
sudo ufw disable                # Ubuntu

# 确认后再打开并正确配置端口
```

### 问题4: 小程序无法连接
- 检查服务器IP是否正确
- 检查防火墙和安全组是否开放5000端口
- 使用 `curl http://服务器IP:5000/api/health` 测试连接
- 微信小程序开发工具需勾选"不校验合法域名"
- 正式环境需要HTTPS和域名备案
